# R0 — overfit-one-batch gate (ANE, real hardware)

**Status: GREEN.** The dense transformer trains from scratch **on the Apple Neural
Engine** and overfits a single pinned batch to near-zero loss. This is the ROADMAP
R0 gate — the first proof that the full ANE loop (forward, backward, optimizer,
weight write-back) is wired correctly, before any ablation number is trusted.

## Config (from `lilbro/configs` → emitted C header)

`r0_overfit`: `dim=64, n_layers=1, n_heads=4, head_dim=16, seq=64, vocab=256,
hidden=128` (MHA, byte vocab). Header generated by the shared config emitter —
the *same* schema that drives the MLX twin:

```bash
python -m lilbro.configs.emit_c r0_overfit > training/training_dynamic/models/gen_r0_overfit.h
```

## Reproduce

```bash
cd training/training_dynamic
python -c "import numpy as np; from lilbro.eval import write_token_stream; \
  write_token_stream('r0_synthetic.bin', np.random.default_rng(0).integers(0,256,128))"
make MODEL=gen_r0_overfit
./train --scratch --overfit --data r0_synthetic.bin --steps 400 --accum 1 --wd 0 --lr 2e-3 --warmup 10
```

`--overfit` pins the batch to a single fixed window (`pos=0`) every step, so the
loss *must* collapse if the loop is correct.

## Result (M3 Max)

| step | 0 | 50 | 100 | 200 | 300 | 390 |
|---|---|---|---|---|---|---|
| loss | 4.645 | 0.738 | 0.188 | 0.054 | 0.023 | **0.0156** |

- **Loss 4.64 → 0.0156** (>290× reduction; starting point is ln(101)≈4.6 because
  vocab compaction maps the 256 byte ids to 101 active tokens).
- Kernel compile: **444 ms** (one-time, 10 dynamic ANE MIL kernels).
- Train: **685 ms / 400 steps = 1.7 ms/step**. Wall: 0.7 s.

## Cross-check vs the fp32 twin

The fp16 ANE floors at ~0.016; the fp32 MLX/torch twins overfit the same-shape R0
config to ~0.010 (`tests/test_overfit.py`). The small gap is consistent with fp16
kernel precision — both collapse decisively, so the ANE loop is behaving.
